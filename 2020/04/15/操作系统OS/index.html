1.2 什么是操作系统

用户角度上 操作系统是一个控制软件

  • 管理应用程序
  • 为应用程序提供服务
  • 杀死应用程序
    内部对象上
  • 资源管理
  • 管理外设、分配资源

操作系统层次结构

  • 硬件之上
  • 应用程序之下（为应用软件提供服务支撑）

linux windows android的界面属于外壳shell，而不是内核kernel，这里只研究kernel，在shell之下

Kernel 操作系统内部组件，包括
CPU调度器
物理内存管理
虚拟内存管理
文件系统管理
中断处理与设备驱动
（uCore操作系统）

OS Kernel的特征（四点）：

  • 并发（并发 一段时间内多个程序同时执行/ 并行 一个时间点上多个程序同时执行）
    • 计算机系统中同时存在多个运行的程序，需要OS管理和调度
  • 共享
    • “同时”访问
    • 互斥共享（分时访问）
  • 虚拟
    • 利用多道程序设计技术，让每个用户都觉得有一个计算机专门为他服务
  • 异步
    • 程序的执行不是一贯到底，而是走走停停，向前推进的速度不可预知
    • 但只要运行环境相同，OS需要保证程序运行的结果也要相同

2.1 操作系统的启动

DISK：存放OS

BIOS——基本I/O处理系统

Bootloader：加载OS

BIOS从一个特定地址开始执行，CS:IP（段寄存器：指令寄存器），POST（加电自检）开始寻找显卡和执行BIOS

bootloader一般放在硬盘的第一个扇区（具体位置由BIOS设置），512字节，主要负责将OS从DISK放入内存

x86为例:

BIOS

  • BIOS将bootloader从硬盘的引导扇区（512字节）加载到0x7c00
  • 跳转到CS:IP=0000:7C00

Bootloader

  • 将OS的代码和数据从硬盘加载到内存
  • 跳转到OS的起始地址
操作系统与设备和程序交互

系统调用、异常、中断

定义：

系统调用（来源于应用程序）system call

  • 应用程序主动向操作系统发出服务请求

异常（来源于不良的应用程序）exception

  • 非法指令或者其他坏的处理状态（如：内存出错）

中断（来源于外设）interrupt

  • 来自不同的硬件设备的计时器和网络的中断

为什么要通过操作系统要来访问外部？

  • 在计算机运行中，内核是被信任的第三方（安全）
  • 只有内核可以执行特权指令（防止访问直接崩溃）
  • 为了方便应用程序（使第三方不用考虑底层细节，提供上层接口）

操作系统如何设计和实现中断、异常、系统调用？

*三者的区别和特点？ *

主要分为源头、处理时间、响应方法三点

源头：

  • 中断：来源于外设
  • 异常：应用程序意想不到的形为
  • 系统调用：应用程序请求操作提供服务

处理时间

  • 中断：异步
  • 异常：同步
  • 系统调用：异步或同步

同步：等待返回

异步：并不等待，保持run状态

响应

  • 中断：持续，对用户应用程序是透明的
  • 异常：杀死或者重新执行意想不到的应用程序指令
  • 系统调用：等待和持续

2.2 操作系统的中断、异常和系统调用

中断和异常的处理过程

硬件处理过程+软件处理过程

1、中断

中断号、异常号，对应特定编号有特定地址，查找到服务例程起始地址

保存与恢复机制

硬件

  • 设置中断标记（CPU初始化）->中断号
    • 将内部、外部事件设置中断标记
    • 中断事件的ID

软件

  • 保存当前处理状态
  • 中断服务程序处理
  • 清除中断标记
  • 恢复之前保存的处理状态

2、异常

  • 异常：异常编号
  • 异常处理
    • 杀死产生异常的程序
    • 重新执行异常指令
  • 恢复现场

3、系统调用

e.g. 应用程序调用printf()，出发系统调用write()

程序访问主要是通过高层次API接口，而不是直接进行系统调用

win32 API for Windows

POSIX API for POSIX-based systems(UNIX LINUX MAC OS X)

Java API for JVM 但是最底层还是win32 or POSIX

  • 需要了解操作系统怎么完成系统调用

图 操作系统与应用程序交互

用户态-内核态

用户态：权限低，无法执行特权和I/O指令

内核态：操作系统可以执行任何一条指令，可以控制整个计算机系统

系统调用时完成用户态到内核态的转换（特权级的转换），系统调用的开销比调用函数开销大，但是更加安全可靠

跨越操作系统边界的开销

  • 在执行时间上的开销超过程序调用
  • 开销：
    • 建立中断/异常/系统调用号与对应服务 例程映射关系的初始化开销
    • 建立内核堆栈
    • 验证参数
    • 内核态映射到用户态的地址空间 更新页面映射权限
    • 内核态独立地址空间 TLB

但是操作系统中进行操作的开销是有必要的，更加安全、可靠
